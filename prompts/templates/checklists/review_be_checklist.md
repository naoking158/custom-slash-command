# Backend Review Checklist

## API Design
- [ ] RESTful 原則準拠 (または GraphQL ベストプラクティス)
- [ ] 適切なHTTPステータスコード
- [ ] 一貫したレスポンス形式
- [ ] バージョニング戦略
- [ ] 適切なエンドポイント命名

## Error Handling
- [ ] 統一されたエラーレスポンス形式
- [ ] 適切なエラーログ出力
- [ ] 機密情報の非露出
- [ ] リトライ可能なエラーの明示

## Data Integrity
- [ ] 入力バリデーション
- [ ] トランザクション管理
- [ ] 楽観的/悲観的ロック考慮
- [ ] データ整合性制約

## Scalability
- [ ] N+1 クエリ問題回避
- [ ] 適切なインデックス設計
- [ ] キャッシュ戦略
- [ ] 非同期処理の適切な使用

## Maintainability
- [ ] 適切なレイヤー分離
- [ ] 依存性注入
- [ ] 設定の外部化
- [ ] ログの適切な粒度

## Testing Considerations
- [ ] ユニットテスト可能な構造
- [ ] モック可能な外部依存
- [ ] テストデータ戦略

# Pull Request Review Checklist

## PR Description
- [ ] 変更の目的が明確に記載
- [ ] 実装アプローチの説明あり
- [ ] テスト方法・確認手順の記載
- [ ] スクリーンショット/動画 (UI変更時)
- [ ] 関連 Issue へのリンク

## Change Analysis
- [ ] 変更範囲が PR タイトルと一致
- [ ] 不要なファイル変更なし
- [ ] 依存関係の追加は妥当
- [ ] マイグレーション手順の明記 (該当時)

## Code Review
- [ ] 既存パターン・規約に準拠
- [ ] エラーハンドリングが適切
- [ ] エッジケースの考慮
- [ ] パフォーマンスへの影響考慮
- [ ] セキュリティ上の懸念なし

## Testing
- [ ] 新規テストの追加
- [ ] 既存テストの更新 (該当時)
- [ ] CI パイプライン通過
- [ ] 手動テスト完了

## Merge Readiness
- [ ] コンフリクト解消済み
- [ ] 必要なレビュー承認取得
- [ ] ドキュメント更新完了
- [ ] リリースノート準備 (該当時)

# Quality Standards

<quality-standards>
<code-quality>
- Follow project coding conventions
- Include appropriate error handling
- Ensure no linting errors
- Ensure no type errors (if applicable)
- Include inline documentation for complex logic
- Cover code with tests as specified
</code-quality>

<code-style>
- Prefer clarity over cleverness
- Use meaningful variable/function names
- Keep functions small and focused
- Handle edge cases explicitly
- Include JSDoc/docstrings for public APIs
</code-style>

<document-quality>
- Address all sections in templates
- Provide specific, actionable content
- Include concrete examples where applicable
- List all assumptions and open questions
- Be self-contained (readable without prior context)
</document-quality>

<verification-checklist>
- [ ] All template sections addressed
- [ ] No placeholder text remaining
- [ ] Examples are concrete and realistic
- [ ] Acceptance criteria are testable
- [ ] Dependencies clearly identified
</verification-checklist>
</quality-standards>
